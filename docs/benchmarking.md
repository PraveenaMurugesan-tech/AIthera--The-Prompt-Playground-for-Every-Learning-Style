# AI Council Benchmarking System

## Purpose
The Benchmarking System evaluates the performance, stability, and consensus quality of the AI Council across various prompts. It allows developers to test how different LLM providers respond, how long they take, and whether the consensus engine is correctly determining a winner based on quality.

## Architecture
The system reuses the existing core architecture without duplicating any logic:
- **`PromptRequest`**: In-memory dummy requests are generated using predefined benchmarks.
- **`LiveCouncil`**: The entire pipeline (`CouncilExecutor`, `ConsensusBuilder`, `PromptScorer`, `ExplanationGenerator`) is invoked for each prompt.
- **`BenchmarkRunner`**: An orchestration script (`scripts/benchmark_council.py`) that loops through the prompts, runs the council, collects statistics, and manages graceful error handling.

## How benchmarking works
1. Loads 25 prompts from `data/benchmark_prompts.json` representing multiple topics, learning styles, and difficulty levels.
2. For each prompt, invokes the AI Council and records the response latency, prompt score, consensus score, success status, and which provider "won".
3. Aggregates data after all runs to determine average latency, fastest/slowest providers, and provider success rates.
4. Generates output reports in JSON, CSV, and Markdown formats.

## Metrics collected
- **Provider responses**: The raw response generated by each provider.
- **Winning provider**: The provider chosen by the consensus engine.
- **Contributors**: The providers whose outputs contributed to the final consensus.
- **Consensus quality score**: The confidence score produced by the `ConsensusBuilder`.
- **Prompt score**: The overall score from `PromptScorer`.
- **Classification**: The quality classification (e.g., GOOD, EXCELLENT).
- **Execution time**: Overall time taken for the prompt request to complete.
- **Individual provider latency**: How long each specific provider took.
- **Provider success/failure**: Did the provider return a valid response?
- **Number of successful providers**: How many providers succeeded in the run.

## How to run
You can execute the benchmarking suite via the command line:

```bash
python scripts/benchmark_council.py
```

## Expected outputs
During execution, progress is printed to the console:
```text
Running benchmark 1/25...
Topic: Python Programming
Learning Style: Step-by-step learner

Execution time: 4.52s
Winner: Gemini
Prompt Score: 85/100
Consensus Score: 0.92
Classification: GOOD
Providers Participated: 3
...
```

At the end of the script, reports are generated in the `reports/` folder:
- **`benchmark_results.json`**: Detailed JSON output including all data for all runs.
- **`benchmark_results.csv`**: A spreadsheet of the runs for easy data analysis.
- **`benchmark_summary.md`**: A summary report of performance, fastests/slowests, and provider frequencies.
